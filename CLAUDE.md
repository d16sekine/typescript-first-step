## Repository Purpose

This is a TypeScript learning repository (typescript-first-step) for beginners, structured as a numbered curriculum from environment setup to modern type features. It's designed for educational purposes; the primary audience is Japanese learners.

## Commands

### Development
- **Install dependencies**: `yarn install`
- **Run the first lesson**: `yarn dev`
- **Run any lesson/exercise directly**: `yarn tsx src/05-narrowing.ts`
- **Type check (src only)**: `yarn type-check`
- **Type check everything (src + tests + exercises)**: `yarn check:all`
- **Run tests**: `yarn test` (watch mode: `yarn test:watch`)
- **Lint**: `yarn lint`
- **Format**: `yarn format` (check only: `yarn format:check`)
- **Compile to JavaScript**: `yarn build` (output in `dist/`)
- **Run compiled JavaScript**: `yarn start`

### Complete verification (what CI runs)
```sh
yarn install --frozen-lockfile
yarn check:all
yarn lint
yarn format:check
yarn test
yarn build
```

## Code Architecture

### Project Structure
- **src/**: Numbered lesson files (`01-first-step.ts` â€¦ `09-type-safety.ts`)
- **exercises/**: Exercise stubs with self-checks; solutions in `exercises/solutions/`; own `tsconfig.json` (relaxed unused-checks so unsolved stubs still compile)
- **tests/**: Vitest tests per lesson; own `tsconfig.json` for type-checking tests
- **dist/**: Compiled JavaScript output (generated by `yarn build`, gitignored)
- **tsconfig.json**: Target ES2022, module commonjs, strict mode, `rootDir: ./src`, `outDir: ./dist`

### Key Configuration
- Uses **yarn classic (v1)** as package manager â€” do not add a `packageManager` field
- **tsx** runs TypeScript files directly without compilation
- **Vitest** is the test framework (`vitest.config.ts`, scoped to `tests/**/*.test.ts`)
- **ESLint** (flat config, `eslint.config.mjs`) + **Prettier** (`.prettierrc.json`, no semicolons)
- CI: `.github/workflows/ci.yml` (Node 20/22/24)

## Conventions
- This is a learning/tutorial repository â€” keep examples simple and educational
- Lesson files are numbered `src/NN-topic.ts`; new lessons continue the sequence
- Code comments and console output are in Japanese
- Every lesson exports its functions/types and guards its demo with:
  `if (typeof require !== "undefined" && require.main === module) { ... }`
  so tests can import lessons without triggering demos (works under tsx and Vitest)
- Exercise stubs use `throw new Error("TODO: ...")` bodies and must pass `yarn check:exercises` unsolved
- Async functions take an optional `delayMs` parameter so tests can pass 0
- README.md is Japanese (primary), README.en.md is English â€” keep both in sync when documenting changes
